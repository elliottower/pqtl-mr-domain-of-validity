## 1. Primary endpoint

**Binary target viability prediction:**

- **Predict SUCCESS** if the target is classified in any causal-positive class (CONCORDANT_CAUSAL, GENETIC_AMPLIFICATION, OBS_INFLATED_CAUSAL).
- **Predict FAILURE** if the target is classified as ZOMBIE or TRUE_NULL.
- **OPPOSITE_DIRECTION** and **INSUFFICIENT** are excluded from the primary analysis. Reported in a sensitivity analysis only.

This is the question that matters for drug development: "Is MR evidence consistent with a causally viable target?"

## 2. V2 taxonomy

Both arms converted to Cohen's d via Chinn (`d = log_OR * sqrt(3)/pi`). Both arms must be in per-SD units before conversion (harmonized from published sources). Thresholds frozen from V1:

- MR null: `|d_MR| < 0.10`
- MR meaningful: `|d_MR| >= 0.10`
- OBS positive: `|d_OBS| >= 0.10`
- OBS null/weak: `|d_OBS| < 0.10`
- Discordance: `Q p < 0.05`

| V2 class | Rule | Clinical prediction |
|---|---|---|
| `CONCORDANT_CAUSAL` | MR meaningful, OBS same direction and meaningful, Q p >= 0.05 | SUCCESS |
| `GENETIC_AMPLIFICATION` | MR meaningful, OBS same direction, Q p < 0.05, \|d_MR\| > \|d_OBS\| | SUCCESS |
| `OBS_INFLATED_CAUSAL` | MR meaningful, OBS same direction, Q p < 0.05, \|d_OBS\| > \|d_MR\| | SUCCESS (lower confidence) |
| `ZOMBIE` | MR null (\|d\| < 0.10), OBS meaningful positive (\|d\| >= 0.10), Q p < 0.05 | FAILURE |
| `TRUE_NULL` | MR null AND OBS null/weak, Q p >= 0.05 | FAILURE |
| `OPPOSITE_DIRECTION` | MR and OBS have opposite signs and at least one arm is meaningful | EXCLUDED (sensitivity: failure) |
| `INSUFFICIENT` | <10 instruments, or failed harmonization, or Egger intercept p < 0.01 (strict) | EXCLUDED |

**Egger flag:** V1 used p < 0.05 as a non-voting diagnostic. V2 uses p < 0.01 as an exclusion criterion for INSUFFICIENT, because horizontal pleiotropy invalidates the IVW estimate that drives all classifications. Targets with 0.01 < Egger p < 0.05 receive a PLEIOTROPIC_POSSIBLE flag but remain scorable.

### 3.2 Inclusion gates (applied deterministically, no manual override)

A target-disease pair enters the V2 screen iff ALL hold:

1. **Exposure GWAS exists** in OpenGWAS with `mr=1` flag.
2. **Instruments:** >= 10 genome-wide significant SNPs at p < 5e-8, LD-clumped locally (r2 < 0.001, 10 Mb window, 1000G EUR reference).
3. **Mean F-statistic >= 10** across instruments.
4. **Observational estimate available:** a published meta-analysis or large prospective cohort reporting an effect estimate with 95% CI for the same exposure-outcome pair. Source chosen by the priority ladder in PRESPEC V1 Part B2 (IPD meta-analysis > random-effects meta-analysis > largest prospective cohort). SE reconstructed from CI only — never 1/sqrt(n).
5. **Non-overlapping samples** where feasible (exposure and outcome GWAS from different cohorts).

### 3.3 Retrospective vs prospective split

- **RETROSPECTIVE:** pivotal Phase III readout already public as of this timestamp. Trial outcome (SUCCESS/FAILURE) is known.
- **PROSPECTIVE:** Phase II+ drug exists but no pivotal Phase III primary endpoint readout. Prediction is locked before readout.
- **NOT SCORABLE:** target has no drug in clinical development for this indication, or outcome is ambiguous/partial.

The sweep CSV's `trial_outcome` field determines the split: SUCCESS and FAILURE are retrospective; PENDING is prospective; EARLY is not scorable for the primary endpoint.

### 3.4 Anti-cherry-pick rules

- The candidate universe cannot be manually edited after hashing.
- No target may be added, removed, or moved between retrospective/prospective after the first V2 Q test is computed.
- If a target fails the instrument gate (< 10 SNPs), it is EXCLUDED, not reclassified.
- The observational source table is frozen with specific PMIDs before scoring. No swapping sources after seeing Q.

## 4. Observational source table

The observational arm is the manual curation bottleneck. For V2, we start with the 13 targets already curated in `data/observational_estimates.csv` and expand by adding published meta-analysis estimates for the 15 FAILURE targets (high priority) and the highest-phase PENDING targets.

Each entry requires:
- PMID of the source meta-analysis
- Effect estimate (OR, HR, or RR) with 95% CI
- Unit (per-SD, per-unit with conversion factor, binary)
- SD conversion if per-unit
- Single source per target per the priority ladder — no swapping

The observational source table is hashed separately before scoring begins.

## 5. Scoring rules

### 5.1 Primary (binary clinical viability)

For each retrospective target-disease pair with a known trial outcome:

- **True positive:** classified causal-positive AND trial_outcome = SUCCESS
- **True negative:** classified null/zombie AND trial_outcome = FAILURE
- **False positive:** classified causal-positive AND trial_outcome = FAILURE
- **False negative:** classified null/zombie AND trial_outcome = SUCCESS

Metrics: sensitivity, specificity, balanced accuracy, PPV, NPV. All reported separately — never a single blended accuracy.

### 5.2 Falsification

- **Primary falsified** if balanced accuracy 90% bootstrap CI lower bound <= 0.5 after >= 30 retrospective evaluable pairs.
- **Prospective falsified** if balanced accuracy 90% bootstrap CI lower bound <= 0.5 after >= 8 prospective readouts.
- Both thresholds inherited from V1 prespec Part C.

### 5.3 Secondary (mechanistic class accuracy)

Descriptive only, not pass/fail. Report confusion matrix of V2 classes vs trial outcomes.

## 9. Relationship to V1

V1 (`PRESPEC_AND_REGISTERED_PREDICTIONS.md`) remains frozen and is the audit record. V1 predictions P1-P3 from Batch 1 and P4-P5/S1-S5 from Batch 2 are scored under V1 rules in `PREDICTION_RESULTS.md`. V2 does NOT retroactively change V1 verdicts.

V2 is explicitly a successor protocol registered after V1 revealed that:
1. The taxonomy lacked a causal-positive discordance class (P4/Lp(a)).
2. The obs-positive threshold correctly excluded weak signals (P5/urate) but the prediction assumed the obs arm would be positive.
3. Binary clinical viability is a more useful primary endpoint than exact mechanistic bucket identity.
